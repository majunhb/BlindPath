# BlindPath 障碍物识别 - 修复说明

## 已完成的修复

### 1. 相机启动问题修复
- **问题**：相机打不开
- **原因**：模型加载失败时直接返回错误，阻止了相机启动
- **修复**：即使模型加载失败，也允许相机启动（演示模式）
- **改进**：添加更详细的错误日志，帮助诊断相机问题

### 2. 错误处理改进
- 改进了 `startDetection()` 方法，模型加载失败不再阻止相机启动
- 改进了 `startCameraSync()` 方法，添加更详细的错误日志
- 确保 `isCameraReady` 状态正确更新

### 3. 代码已推送
- 提交哈希：dc29ae9
- GitHub Actions 正在构建新 APK

## 待解决的问题：AI 模型文件

### 问题
`yolov8n.tflite` 模型文件不在 `app/src/main/assets/` 目录中，导致：
- AI 检测返回空列表（"没有内容一样的"）
- 应用运行在"演示模式"

### 解决方案

#### 方案 1：手动下载并放置模型文件（推荐）

1. **下载 YOLOv8n 模型**：
   - 从 Ultralytics 官网下载：https://docs.ultralytics.com/
   - 或者从 Hugging Face 下载预转换的 TFLite 模型
   - 文件名：`yolov8n.tflite`

2. **放置模型文件**：
   - 将 `yolov8n.tflite` 文件复制到：`BlindPath/app/src/main/assets/` 目录
   - 如果没有 `assets` 目录，请创建它

3. **重新构建 APK**：
   ```bash
   cd BlindPath
   ./gradlew assembleDebug
   ```

#### 方案 2：使用演示模式（临时方案）

即使没有模型文件，应用也能：
- 启动相机
- 正常运行（但不进行 AI 检测）
- 用于测试相机功能和 UI

## 精度优化建议

### 当前参数
- 置信度阈值：`0.5` (AIDetector.kt:119)
- IoU 阈值：`0.45` (AIDetector.kt:120)

### 优化建议
1. **降低置信度阈值**（提高召回率）：
   - 修改为 `0.3` 或 `0.4`
   - 可能会检测到更多物体，但可能有更多误报

2. **调整 IoU 阈值**（优化 NMS）：
   - 修改为 `0.5` 或 `0.6`
   - 减少重叠检测框的合并

3. **距离估算优化**：
   - 校准焦距 (`calibratedFocalLength`)
   - 根据实际摄像头参数调整

## 故障排除

### 相机仍然打不开
1. 检查相机权限是否授予
2. 确认其他应用没有占用相机
3. 重启手机后重试
4. 查看日志：`adb logcat | grep BlindPath`

### 识别精度不高
1. 调整置信度阈值（见上文）
2. 确保光照充足
3. 避免强光或逆光环境
4. 保持摄像头清洁

### 语音播报问题
1. 检查 TTS 引擎是否安装
2. 调整 TTS 语速和音调
3. 确认媒体音量不是静音
